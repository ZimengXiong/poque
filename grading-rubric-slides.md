# Poker Game Presentation Rubric

60-point grading guide for evaluating poker architecture, AI collaboration, responsible AI, and presentation skills.

---

# Scoring Scale

| Points | Level | Meaning |
|---:|---|---|
| 4-5 | Advanced | Exceeds expectations; exceptionally clear, detailed, and insightful. |
| 3 | Proficient | Meets all requirements completely and competently. |
| 1-2 | Developing | Partially addresses the criterion but lacks depth, clarity, or completion. |
| 0 | Not Demonstrated | Missing entirely. |

# Game Logic & OOP

Score: ____ / 5

Look for:

- Clear explanation of cards, decks, players, chip pools, and betting rounds
- Clean programming principles: classes, methods, loops, and separation of concerns
- Logic that can be followed without guessing how the game state changes

---

# Variant-Specific Rules & Hand Rankings

Score: ____ / 5

For Texas Hold'em:

- Community cards are implemented
- Flop, turn, and river structure is correct
- 5-out-of-7 hand evaluation is accurate

For 5-Card Draw:

- Discard and draw phase works
- Deck depletion is handled
- Hand evaluation is accurate

---

# Live Demo + Code Readability

Live Demonstration / Execution: ____ / 5

- Working demo runs smoothly
- Bugs are handled or explained gracefully
- Presenter can troubleshoot clearly

Code Readability & Documentation: ____ / 5

- Snippets are clean, legible, and well-commented
- Variable names and function logic are explained
- Slides avoid crowded code blocks

# Prompting Strategy & Iteration

Score: ____ / 5

Look for:

- Specific prompt examples, not vague claims
- Evidence of iteration after broken, outdated, or inefficient code
- Clear explanation of how prompts improved the final implementation

---

# Verification & Debugging

Score: ____ / 5

Look for:

- Students actively verified AI output instead of blindly trusting it
- They caught hallucinations or logic errors
- They tested tricky poker cases like high-card kickers, ties, and split pots

---

# AI Contribution Breakdown

Score: ____ / 5

Look for:

- Clear distinction between student-written and AI-generated work
- Explanation of what AI generated, optimized, or refactored
- Honest discussion of where students made final technical decisions

# Academic Integrity

Attribution & Documentation: ____ / 5

Look for:

- Inline comments or slides acknowledging AI-generated snippets
- Clear citation of AI assistance
- No attempt to pass off AI code as entirely original thinking

---

# Fairness & Bot Design

Algorithmic Bias & Strategic Fairness: ____ / 5

Look for:

- Discussion of how AI training data may affect behavior
- If using a bot, confirmation that it does not cheat
- Bot decisions based only on visible game state or math, not hidden cards

---

# Broader AI Implications

Score: ____ / 5

Look for:

- Technical limitations of generative tools
- Intellectual property or training-data concerns
- Environmental costs of LLM usage
- Risks of relying on generated code without understanding it
